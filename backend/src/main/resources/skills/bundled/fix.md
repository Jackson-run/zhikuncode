# /fix — 智能修复

## 目标
根据错误信息或失败的测试，诊断并修复当前项目中的错误。

## 步骤
1. 如果提供了错误信息，分析它以识别根本原因
2. 如果没有提供错误信息，运行 `git diff` 检查可能引入问题的近期变更
3. 使用错误位置（文件、行号、堆栈跟踪）搜索相关文件
4. 识别根本原因：
   - 语法错误
   - 类型不匹配
   - 缺少导入或依赖
   - 逻辑错误
   - 配置问题
5. 应用能够解决问题的最小修复
6. 通过运行相关测试或构建命令验证修复效果

## 规则
- 始终在应用修复之前识别根本原因
- 优先选择最小化、有针对性的修复，而非大规模重构
- 如果发现多个问题，逐个修复
- 说明问题的原因以及修复为何有效
- 修复后运行测试以验证没有引入回归
