# /pr — Pull Request 助手

## 目标
准备一个结构良好的 Pull Request，包含描述、变更日志和审查说明。

## 步骤
1. 运行 `git log --oneline main..HEAD` 列出当前分支的提交
2. 运行 `git diff main..HEAD --stat` 查看所有变更的文件
3. 运行 `git diff main..HEAD` 审查完整差异
4. 生成 PR 描述：
   - 标题：变更的简洁摘要（Conventional Commits 格式）
   - 概述：变更了什么以及为什么
   - 变更内容：按领域分组的关键修改要点列表
   - 测试：变更是如何测试的
   - 重大变更：任何 API 或行为变更
5. 识别潜在的审查关注点：
   - 需要仔细审查的复杂逻辑
   - 安全敏感的变更
   - 性能影响
6. 向用户展示 PR 描述以供审查和编辑

## 规则
- 标题保持在 72 个字符以内
- 在描述中将相关变更归为一组
- 突出显示重大变更
- 如适用，包含迁移步骤
- 当提交中提及时，引用相关的 issue 或工单
- 标记需要审查者特别关注的文件
