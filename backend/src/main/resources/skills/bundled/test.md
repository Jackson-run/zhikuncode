# /test — 智能测试

## 目标
为指定代码或近期变更生成或运行测试。

## 步骤
1. 确定需要测试的内容：
   - 如果指定了文件/函数，测试该目标
   - 如果没有指定目标，分析 `git diff` 查找近期变更的代码
2. 确定使用的测试框架（JUnit、Vitest、Jest 等）
3. 生成测试时：
   - 分析目标代码的公共 API
   - 生成覆盖以下场景的测试用例：正常路径、边界情况、错误情况
   - 遵循项目中已有的测试模式
4. 执行测试时：
   - 运行相关的测试命令
   - 报告结果：通过、失败、跳过
   - 对于失败的测试，分析原因并建议修复方案

## 规则
- 遵循项目已有的测试约定和模式
- 测试公共行为，而非实现细节
- 包含边界情况：null/空输入、边界值、错误条件
- 使用描述性的测试名称，说明被测试的内容
- 适当地模拟外部依赖
- 追求有意义的覆盖率，而非 100% 的行覆盖率
